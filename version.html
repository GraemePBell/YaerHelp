Version History

Version 2.0.04.0

Minor feature release.

  • Added pictorial tooltips back to into the Library Views.
  • Stability fixes.
  • Fix for very rare, but not rare enough, crash on right-click due to WinUI's more aggressive virtualisation of ListView controls. It now handles the event more gracefully. Commentary - Why does it recycle a control when it's just been interacted with? That seems a bit *too* aggressive.
  • Fix for issue with umlauts in file paths.
  • Fix for WebView2 not loading UTF8 reliably if not explicitly specified with a BOM.

Version 2.0.03.0

Minor feature release.

  • Added a Library Refresh button. Fixes the rare issue of the library not updating correctly after adding or removing books.
  • Added checks for some nulls that should never be null.
  • Added andling of some rare cases of missing or malformed files in ePubs - handle html files missing </head> tags more gracefully than they deserve.
  • Added more C++23 features to improve performance and safety.
  • Better Contents Page handling.

Version 2.0.02.0

Minor bugfix release.

  • Better handling of dropped items during decompression.
  • Better Dialog boxes for removing books and for exceptions.
  • Fix for exceptions during decompression not clearing the progress control and restoring the import button.
  • Better handling of empty library search results.

Version 2.0.01.0

Major release with significant updates and improvements.

  • Complete overhaul of the Codebase. Now using C++23, WebView2 and the most recent, stable, version of the Windows APP SDK. YaerZero is now a fully modern WinUI application.
  • Performance and safety improvements from to upgrading from C++17 to C++23 and WebView2.
  • Minor UI/UX changes from using WinUI controls, a new logo and updated icons.

Version 1.0.23.0

Bug fixes and performance improvements, including:

  • Bug Fix for extremely long cover file names in ePubs.
  • Mitigation for erroneous</img> tags. The HTML specification defines the <img> tag as a "void" element, which means it cannot have any content and must not have an end tag. However, some ePubs do include an end tag for the element, even though this is not valid according to the specification.
  • Mitigation for a bad or missing MIMETYPE file in otherwise valid ePubs.
  • Added the ability to play embedded video files, if supported by the embedded browser.
  • Many small stability, reliability and portability fixes.
  • Updated to Boost 1.91.
