Version History

Version 2.0.08.0

Minor bug-fix release.

  • Speed, reliability and "quality-of-life" improvements.

Commentary:

  • As a programmer who likes to get down in the weeds, so-to-speak, that kind of utterly uninformative "release notes" is an, admittedly minor, irritation. Stuff changed, nothing I can explain without getting more technical than an average user wants to know.

    I'm not an average user - maybe you're not either.

    Some CSS management fixes, re-organising the code that parses and regularises incoming CSS.

    Continuing to replace std::wstring and local string literals with constexpr std::wstring_view. Likewise, replacing std::vector with constexpr std::array, wherever the size is fixed and known at compile time.

    Fixed a problem with one particular epub that caused the program to hang on thumbnail generation. Still investigating the root cause, but the result was a duplication in the list of thumbnails to be generated and the code was waiting on an image that would never appear. De-duplicated the std::vector, while still trying to figure out why the code would write twice, just for that one book. It's all very strange, but the bodge is in place and the program no longer hangs on that book. The root cause will be found and fixed in a future release.

    Gathered together all the string literals used as file extensions, moved them into a single header file and made them constexpr std::wstring_view. When in the heat of coding it's too easy to type ".jpg" rather than a consistent constant name (wide_JPG_ext, u8_JPG_ext, narrow_JPG_ext, or whatever). Or ".jpg" in one place and ".jpeg" in another, or ".cbz" and ".zip". Or worse, ".jpg" and ".JPG". I found no errors, but now it's all in one place and consistent.

    Fixed an edge-case in content page handling, where the old content page from a previous book was being held and overwriting the new value. It required a series of steps to reproduce that was very rare in the UWP version (effectively unreproducible because UWP just does the right thing when marshalling data across the ABI), but it became slightly more likely in the WinUI version where I have to mimic the UWP marshalling behaviour.

    Handled the fallback case of a missing cover image. This was a solved problem in the EdgeHTML version, but WebView2 is a bit more awkward about HTML reading from the APP package. The simplest solution was, if a book has no cover, to copy the fallback cover image, from the APP package, into the book images folder and reference it from there.

    Displaying ampersands in book titles regressed. Now fixed.

Version 2.0.07.0

Minor feature release.

  • Thumbnailing of images now explicitly uses up to 15 parallel threads - or hardware threads minus one - whichever is fewer. This results in a significant speedup of Comic Book processing on modern, multicore, CPUs. The previous implementation based on the UWP version serialised output as thumbnailing was very much faster on UWP.

Bug Fix.

  • Regression of handling internal covers in merged epubs.

Version 2.0.06.0

Minor feature release.

  • Added support for deleting books from the library, most recent list and the panel, using the keyboard shortcut Ctrl+D.
  • Additional UI feedback on long-running delete operations.
  • Progress bar for long-running import operations.
  • Small code optimisations with C++20/23 language features.

In-wild Bug Fixes - both caused by the WebView2 chromium engine having some interesting CSS edge cases. EdgeHTML either ignored or handled these cases better. Can always rely on legitimate publishers to push the boundaries of well-formed content. In these cases, both book publishers had used preprocessors to generate final the CSS and they left artifacts behind. I have opinions on this.

  • Fix for black on black text in a book due to complex CSS in the header style element.
  • Fix for a full WebView2 hang when opening an epub with complex and somewhat malformed CSS, stored in a CSS file, interacted with YaerZero's own layout, causing WebView2 to go a bit infinite-loopy.

Version 1.0.23.0

Bug fixes and performance improvements, including:

  • Bug Fix for extremely long cover file names in ePubs.
  • Mitigation for erroneous</img> tags. The HTML specification defines the <img> tag as a "void" element, which means it cannot have any content and must not have an end tag. However, some ePubs do include an end tag for the element, even though this is not valid according to the specification.
  • Mitigation for a bad or missing MIMETYPE file in otherwise valid ePubs.
  • Added the ability to play embedded video files, if supported by the embedded browser.
  • Many small stability, reliability and portability fixes.
  • Updated to Boost 1.91.
