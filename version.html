Version History

Version 2.0.07.0

Minor feature release.

  • Thumbnailing of images now explicitly uses up to 15 parallel threads - or hardware threads minus one - whichever is fewer. This results in a significant speedup of Comic Book processing on modern, multicore, CPUs. The previous implementation based on the UWP version serialised output as thumbnailing was very much faster on UWP.

Bug Fix.

  • Regression of handling internal covers in merged epubs.

Version 2.0.06.0

Minor feature release.

  • Added support for deleting books from the library, most recent list and the panel, using the keyboard shortcut Ctrl+D.
  • Additional UI feedback on long-running delete operations.
  • Progress bar for long-running import operations.
  • Small code optimisations with C++20/23 language features.

In-wild Bug Fixes - both caused by the WebView2 chromium engine having some interesting CSS edge cases. EdgeHTML either ignored or handled these cases better. Can always rely on legitimate publishers to push the boundaries of well-formed content. In these cases, both book publishers had used preprocessors to generate final the CSS and they left artifacts behind. I have opinions on this.

  • Fix for black on black text in a book due to complex CSS in the header style element.
  • Fix for a full WebView2 hang when opening an epub with complex and somewhat malformed CSS, stored in a CSS file, interacted with YaerZero's own layout, causing WebView2 to go a bit infinite-loopy.

Version 2.0.05.0

Minor bug-fix release.

  • Stability fix.
  • Fix for an issue in NCX with single-quotes in navPoints. I'm not sure why this suddenly started causing issues, since the code and the test files were unchanged. I suspect a recent change in Regex behaviour, making it vulnerable in edge cases.

Version 1.0.23.0

Bug fixes and performance improvements, including:

  • Bug Fix for extremely long cover file names in ePubs.
  • Mitigation for erroneous</img> tags. The HTML specification defines the <img> tag as a "void" element, which means it cannot have any content and must not have an end tag. However, some ePubs do include an end tag for the element, even though this is not valid according to the specification.
  • Mitigation for a bad or missing MIMETYPE file in otherwise valid ePubs.
  • Added the ability to play embedded video files, if supported by the embedded browser.
  • Many small stability, reliability and portability fixes.
  • Updated to Boost 1.91.
